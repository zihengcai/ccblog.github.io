单体架构设计与项目开发（一）

基于Springboot2.x实现单体架构设计与准备工作

  1. 前后端主要技术选型

  2. 前后端分离开发模式

  3. 后端分层架构模式

  4. 数据库表设计

  5. Springboot Spring MyBatis 框架整合（SSM）

技术选型

Springboot vs SpringMVC

  1. SpringMVC

    框架，是spring的一个子集/web模块；

  2. Springboot

  • 工具，配置整合工具，通过pom依赖，实现了自动配置；
  • 从配置繁琐xml到零配置（yml）
  • 集成了多样化中间件（*-starter）
  • 从外置tomcat变为内置tomcat

Struts：为什么不用？有安全漏洞；action线程安全问题，请求变量共享；Filter拦截；非spring自产自销

前端技术选型

  • MVVM开发模式
  • Jquery、vue.js(渐进式，周期性迭代替代)
  • html
  • css

技术选型所要考虑的

  • 切合业务
  • 社区活跃度
  • 团队技术水平：熟悉？陌生？新技术？旧技术？团队培训
  • 版本更新迭代周期：技术的更新频次
  • 试错精神：备选方案
  • 安全性
  • 成功案例
  • 开源精神：是否有闭源的可能性

前后端分离开发模式

早期传统javaweb开发模式


单体项目中，mvc架构后端代码及部分前端代码统计打包在war包中，其中html代码是由jsp渲染而来的，渲染的过程是在服务器端中做的，当用户量越来越多，服务器会有非常大的性能影响。

前后端单页面交互，mvvm开发模式


浏览器访问静态页面h5页面，nginx反向代理作为静态资源服务器，用户请求至后台服务器（动态请求），通过restful返回json格式数据。

手机端也可以作为浏览器进行这样的访问，除此之外还有客户端去请求同样的内容，后端的api根据客户端做一些微调。

这样后端代码只需要一套就可以了，可以明确分工，并行开发。并且可以精准定位bug。
缺点：沟通；发布。

这也是一种架构模式，前端与后端的项目是不同的项目，两套服务器两套代码，分别部署，不同的开发团队进行开发维护。

项目拆分与聚合


项目通过maven进行聚合。
最终运行的是一个war包或jar包，通过右面的子模块聚合形成。

pojo.jar:一些实体类

mapper.jar:数据层

service.jar:业务层

controller.jar:处理请求

通过pom进行依赖引用，不需要重新开发。一个组件可以被多个项目进行依赖。

聚合工程的构建是一个复杂的过程，会单独使用一篇新的文章进行记录。
